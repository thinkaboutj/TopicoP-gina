FTP Server

FTP (File Transfer Protocol)

Es un protocolo de red utilizado para transferir archivos entre un cliente y un servidor a través de una red TCP/IP, como Internet. Permite subir, descargar, renombrar y eliminar archivos de manera remota, y es ampliamente usado en entornos corporativos, servidores web y administración de sistemas.

Implementacion

Se implementó un servidor FTP gratuito utilizando InfinityFree, un servicio de hosting web que permite subir y gestionar archivos mediante el protocolo FTP. Utilizando la app FTPManager desde un iPhone, se logró establecer una conexión remota con el servidor, cargar archivos en la carpeta pública (htdocs) y acceder a ellos desde cualquier navegador web mediante una URL personalizada. Esta solución demuestra cómo transferir archivos fácilmente entre dispositivos usando una plataforma gratuita y accesible.

MySQL Server

Instalación, configuración y optimización de bases de datos relacionales para un almacenamiento estructurado y seguro.A continuación, presentamos un video explicativo sobre el servicio MySQL, donde podrás conocer más sobre el trabajo realizado.

Para la realización de este servicio se llevaron a cabo los siguientes pasos:

  • Primero, se instaló el servidor de base de datos con el comando sudo apt install mysql-server.
  • Posteriormente, se modificó el archivo /etc/mysql/mysql.conf.d/mysqld.conf para cambiar la opción bind-address a 0.0.0.0, permitiendo conexiones desde cualquier dirección IP (IPv4 e IPv6).
  • Luego, se instaló el servicio web Apache2, el cual sirvió como entorno para un formulario web desarrollado con PHP, JavaScript y HTML.
  • Este formulario permite ingresar datos que se almacenan directamente en la base de datos MySQL.
  • Finalmente, se accedió al servidor MySQL utilizando el comando sudo mysql -u root -p para verificar que la información del formulario se almacenara correctamente en las tablas.

Servidor Git

Este servicio consiste en la instalación, configuración y despliegue de un servidor Git privado utilizando contenedores Docker sobre un sistema operativo Ubuntu Linux. Su propósito principal es brindar a equipos de desarrollo o usuarios individuales la posibilidad de alojar y gestionar sus propios repositorios de código de manera autónoma, sin depender de servicios externos como GitHub, GitLab o Bitbucket.

Pasos para levantar un servidor Git usando Docker en WSL (Ubuntu)

  1. 1. Instalación de Docker en WSL

    Primero, asegúrate de tener Docker instalado en tu sistema WSL con Ubuntu. Si no lo tienes, sigue estos pasos:

    
# Actualiza tu lista de paquetes
sudo apt update

# Instala las dependencias necesarias
sudo apt install apt-transport-https ca-certificates curl software-properties-common

# Añade la clave GPG de Docker
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

# Añade el repositorio de Docker
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"

# Actualiza nuevamente la lista de paquetes
sudo apt update

# Instala Docker
sudo apt install docker-ce

# Verifica la instalación
sudo systemctl status docker

    Una vez instalado Docker, asegúrate de que el servicio Docker se esté ejecutando.

  2. 2. Instalación de Git en el contenedor

    Ahora, crearemos un contenedor Docker con Git instalado:

    
# Crea un contenedor con Ubuntu
sudo docker run -it ubuntu bash

# Instala Git dentro del contenedor
apt update
apt install git
  3. 3. Configuración del repositorio Git

    Una vez instalado Git dentro del contenedor Docker, configura tu nombre y correo electrónico:

    
# Configura tu nombre
git config --global user.name "Tu Nombre"

# Configura tu correo electrónico
git config --global user.email "tuemail@example.com"
  4. 4. Crear un repositorio Git en el contenedor

    Crea una carpeta de repositorios y un repositorio nuevo dentro del contenedor Docker:

    
# Crea un directorio para tus repositorios
mkdir /repositorios
cd /repositorios

# Inicializa un nuevo repositorio
git init mi-repositorio
  5. 5. Acceder al repositorio desde tu máquina local

    Puedes compartir este repositorio con tu máquina local utilizando SSH o por medio de un volumen de Docker. Asegúrate de exponer el puerto necesario para acceder al repositorio desde cualquier lugar.

  6. 6. Levantar el servicio de Git

    Si deseas mantener el servidor Git corriendo permanentemente, puedes hacerlo con Docker Compose. Aquí un ejemplo:

    
# Crea un archivo docker-compose.yml
version: '3'
services:
  git-server:
    image: ubuntu
    container_name: git-server
    command: bash -c "while true; do sleep 1000; done"
    ports:
      - "8080:8080"

    Este archivo mantendrá el contenedor en ejecución y disponible para aceptar conexiones a través del puerto 8080.

  7. 7. Clonar el repositorio Git

    Desde tu máquina local, puedes clonar el repositorio Git que se encuentra en tu servidor Docker usando el siguiente comando:

    
git clone http://localhost:8080/mi-repositorio
